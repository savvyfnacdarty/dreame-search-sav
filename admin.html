Ce guide explique comment mettre à jour le catalogue de deux façons : en ajoutant des produits manuellement ici, ou en mettant à jour le fichier SharePoint automatique. Dans les deux cas, un commit GitHub est nécessaire pour que les changements soient visibles par tous les utilisateurs.
Méthode 1
Ajout manuel d'un produit
Idéal pour un ou deux produits — tout se fait ici, pas besoin de SharePoint
1
Créer le produit dans l'onglet Catalogue
Cliquez sur + Nouveau produit, saisissez le nom du dossier exactement comme il apparaît dans SharePoint (ex : P001 - Dreame P70 Pro Ultra), puis collez l'URL du dossier SharePoint.
2
Trouver le lien d'un fichier SharePoint
Dans SharePoint, faites un clic droit sur le fichier → Copier le lien. Le lien ressemble à : https://fnacdarty.sharepoint.com/:x:/r/sites/…/BOM_P70.xlsx?… Collez-le tel quel dans le champ URL — le site ajoutera automatiquement ?download=1 pour forcer le téléchargement.
3
Ajouter les fichiers et sauvegarder
Ajoutez autant de fichiers que nécessaire (BOM, PDF, Word…), choisissez le type dans le menu déroulant, puis cliquez sur Enregistrer.
4
Exporter catalog.json
Cliquez sur Exporter catalog.json en haut de la page. Un fichier se télécharge dans votre dossier Téléchargements.
5
Commiter le fichier sur GitHub
Allez sur github.com → votre repo → dossier data/ → cliquez sur catalog.json → icône crayon ✏️ → Upload files ou collez le contenu → cliquez Commit changes.

⏱ Dans 1 à 2 minutes, le site est mis à jour pour tous les utilisateurs.
Méthode 2
Mise à jour automatique via SharePoint
Idéal pour mettre à jour tout le catalogue d'un coup — export direct depuis SharePoint
1
Exporter l'arborescence SharePoint
Dans SharePoint, naviguez vers le dossier racine des documents SAV DREAME → cliquez sur ⚙ Exporter vers Excel (ou Export to Excel) dans la barre d'outils SharePoint.
Un fichier .xlsx se télécharge avec toutes les colonnes Nom, Chemin d'accès, Type d'élément, Modifié.
2
Renommer le fichier
Renommez le fichier téléchargé exactement : extraction_sharepoint.xlsx
3
Commiter sur GitHub à la racine du repo
Sur GitHub → votre repo → racine → Add file → Upload files → déposez extraction_sharepoint.xlsx → Commit changes.
Le fichier doit être à la racine du repo, pas dans un sous-dossier.
4
Vérifier sur le site
Attendez 1-2 minutes puis testez une recherche sur index.html. Si un dossier n'apparaît pas, vérifiez que son chemin contient bien un identifiant projet (ex: P001) ou un tiret.
📋 Structure du catalog.json Pour référence technique
Le fichier catalog.json suit cette structure. Vous pouvez aussi l'éditer manuellement dans un éditeur de texte si besoin.
{
  "produits": [
    {
      "id": "p001",
      "dossier": "P001 - Dreame P70 Pro Ultra",
      "url_dossier": "https://fnacdarty.sharepoint.com/sites/.../P001",
      "fichiers": [
        {
          "nom": "BOM Pièces détachées",
          "url": "https://fnacdarty.sharepoint.com/.../BOM_P70.xlsx",
          "type": "xlsx"
        },
        {
          "nom": "Guide SAV",
          "url": "https://fnacdarty.sharepoint.com/.../Guide_P70.pdf",
          "type": "pdf"
        }
      ]
    }
  ]
}
