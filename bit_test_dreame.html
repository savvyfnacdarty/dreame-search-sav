Guide opératoire — Test BIT.

Module de formation pour les techniciens SAV PEM. Référence d'atelier pour exécuter, après intervention, les procédures de test BIT sur les robots aspirateurs Dreame & MOVA. À garder ouvert sur le banc.

▸ §01 Préparation Prérequis avant intervention

Avant de toucher au robot.

Trois prérequis conditionnent l'accès au mode test BIT. Sans eux, l'interface de calibrage n'est pas visible dans l'application.

01.1

Conditions préalables

01
Application installée App
Un téléphone avec l'application Dreamehome ou Mi Home installée et connectée au robot.
02
Whitelist de calibrage activée Attention
Contacter le support technique pour activer la whitelist de calibrage sur Dreamehome / Mi Home pour le compte utilisé. Sans cette autorisation, les options de test et calibrage restent masquées.
03
Firmware à jour App
Mettre à jour le firmware du robot vers la dernière version disponible.
⚠ Test BIT par composant unique
Les trois conditions ci-dessus doivent toutes être réunies pour effectuer le test BIT par composant unique via l'app. Le test séquentiel offline (§02.3) ne nécessite que le firmware à jour.
01.2

Notes importantes

⓵ Après réparation
Un test BIT est obligatoire après chaque réparation pour vérifier le fonctionnement normal du robot.
⓶ Avant réparation
Effectuer un dépannage du robot avant l'intervention permet d'identifier précisément le ou les composants défectueux et d'éviter les démontages inutiles.
▸ §02 Test BIT Built-in test Diagnostic composants

Test BIT — diagnostic composant par composant.

Deux modes complémentaires : test ciblé via l'app pour un composant suspect, ou test séquentiel offline pour une vérification complète après réparation.

§02.1 — §02.2
Test ciblé via app
Accès à l'écran de test BIT dans Mi Home / Dreamehome. On choisit un composant à la fois. Idéal pour valider une réparation localisée.
§02.3
Test séquentiel offline
12 étapes enchaînées sur le robot, sans application. Le robot guide vocalement le technicien. Idéal pour une revue complète post-intervention.
§02.4
Autres tests via app
Front window, TOF, géomagnétique. Tests automatiques avec retour vocal, accessibles uniquement depuis l'écran BIT de l'app.
02.1

Accès au mode test BIT via l'application

01
Ouvrir Mi Home et se connecter au robotApp
Sur la page d'accueil du robot, cliquer sur les trois points en haut à droite pour entrer dans l'interface de paramètres Settings.
02
Entrer dans l'historique de nettoyageApp
Cliquer sur Cleaning history.
03
Déverrouiller les informations réseauAttention
Cliquer 10 fois rapidement sur la zone des compteurs (Total time / Total area / Total times) en haut de l'historique. L'écran 网络信息 (Network info) apparaît.
04
Ouvrir BIT CheckApp
Dans la liste des infos réseau, cliquer sur BIT Check.
05
Confirmation d'entrée en mode BITOK
L'écran BIT 单器件检测 (détection BIT par composant) s'affiche. Le robot est prêt pour les tests ciblés.

02.3

Test BIT séquentiel — 12 étapes, sans application

Le robot exécute une inspection complète dans l'ordre ci-dessous, avec guidage vocal. Aucune connexion à l'app n'est requise.

Entrée en mode séquentiel
Dans les 3 minutes suivant l'allumage : maintenir le bouton ⌂ Recharge ENFONCÉ et appuyer 5 fois sur ⏻ Power → le robot entre en mode test séquentiel.
Manuel  Action requise du technicien Auto  Test automatique, attendre le bip / la voix Vocal  Résultat annoncé par le robot
01
Auto-diagnostic systèmeManuelVocal~30 s d'attente
Appuyer sur Bouton 1 — Power pour démarrer : le robot entre en auto-diagnostic système.
Quand le robot annonce « Veuillez placer le robot dans la position spécifiée et cliquer sur le bouton 2 pour démarrer le test », appuyer sur Bouton 2 — Recharge.
02
Test pare-chocsManuel
Appuyer puis relâcher le pare-chocs gauche, puis appuyer et relâcher le pare-chocs droit.
03
Test capteur de chuteManuel
Soulever le robot du sol pour déclencher les capteurs de chute (cliff sensors).
04
Test capteur de bord (PSD)Manuel
Couvrir les capteurs de bord situés sur les côtés du robot (avec la main ou un objet opaque).
05
Test bumper LDSManuel
Appuyer puis relâcher le cache gauche du LDS (haut de la tour LDS, côté gauche), puis le cache droit.
06
Test roues motricesAutoVocal~15 s
Le robot entre et termine le test automatiquement. Attendre le bip de confirmation.
07
Test gyroscopeAutoVocal~10 s
Test automatique. Attendre le retour vocal du robot.
08
Test bac à poussièreManuel
Sortir le bac à poussière et le remettre en place.
09
Test réservoir d'eauManuelAttention
Sortir le réservoir et le remettre en place.
⚠ Selon modèle — réservoir intégré non démontable
Les positions des réservoirs varient. Sur les modèles avec base de remplissage automatique, le réservoir d'eau est généralement intégré au robot et non démontable : ne pas effectuer cette étape.
  • Modèles à station + réservoir démontable (étape OK) : MOVA E20, gammes L-series où le bac eau se clique séparément.
  • Modèles à station + réservoir intégré non démontable (passer l'étape) : la plupart des séries Ultra/Master récentes avec base auto-fill (X50 Ultra Complete, X50 Master, série Aqua10, L20 Ultra, Matrix10 Ultra…).

Liste à compléter par l'atelier — vérifier physiquement sur le robot avant manipulation. Soumettre les corrections via OVPEM.

10
Test turbine d'aspirationManuelAutoVocal~15 s
Placer le robot au sol, dessous orienté vers le bas. Après l'annonce vocale : la turbine démarre, puis s'arrête. Le résultat est annoncé par voix.
11
Test brosse latéraleManuelAutoVocal~15 s
Robot dessous vers le bas. Après l'annonce vocale, la brosse latérale tourne dans les deux sens à différentes vitesses, puis s'arrête. Résultat vocal.
12
Test brosse principaleManuelAutoVocal~15 s
Robot dessous vers le bas. Après l'annonce, la brosse principale tourne dans les deux sens à différentes vitesses, puis s'arrête. Résultat vocal.
✓ Fin du test séquentiel
Après l'étape 12, le test BIT séquentiel est terminé. Si un test a échoué, identifier le composant et reprendre le diagnostic ciblé via §02.1.
02.4

Autres tests via l'application BIT

Ces trois tests sont accessibles uniquement depuis l'écran BIT de l'app. Exécution automatique avec retour vocal.

01
Test fenêtre avantAppAutoVocal
Sur l'écran BIT, cliquer Front window test. Après l'annonce vocale, le robot démarre l'auto-diagnostic. Résultat annoncé en fin de test.
02
Test TOFAppManuelVocal
Cliquer TOF test. Après l'annonce vocale, bloquer la fenêtre TOF à l'avant du robot puis la dégager. Résultat annoncé.
⚠ Selon modèle
Certains modèles n'ont pas de capteur TOF — ne pas exécuter ce test sur ces modèles.
03
Test géomagnétiqueAppAutoVocal
Cliquer Geomagnetic Test. Auto-diagnostic, puis résultat vocal.
